# 기존 스킬 업데이트 가이드

sync-skills의 Step 5에서 기존 verify-* 스킬을 업데이트할 때 참조합니다.

## 핵심 규칙

- **추가/수정만** — 아직 작동하는 기존 검사는 절대 제거하지 않음
- **Related Files** 테이블에 새 파일 경로 추가
- 변경된 파일에서 발견된 패턴에 대한 새 탐지 명령어 추가
- 커버되지 않은 규칙에 대한 새 워크플로우 단계 추가
- 코드베이스에서 삭제가 확인된 파일의 참조 제거
- 변경된 특정 값(식별자, 설정 키, 타입 이름) 업데이트

## Related Files 추가 예시

```markdown
## Related Files

| File | Purpose |
|------|---------|
| ... 기존 항목 ... |
| `<새로운 파일 경로>` | <이 파일의 역할 설명> |
```

## 탐지 명령어 추가 예시

````markdown
### Step N: <검증 대상 설명>

**목적:** <왜 이 검사가 필요한지>

**방법:**

```bash
Grep: pattern=`<정규식>` path=<경로> glob=<패턴>
```

**PASS:** <통과 조건>
**FAIL:** <실패 조건 + 수정 방법>
````

## 워크플로우 단계 추가 시 규칙

1. 기존 Step 번호를 유지하고, 새 단계는 마지막에 추가
2. 각 단계에 반드시 PASS/FAIL 기준 명시
3. 실패 시 수정 방법(코드 예시 포함) 제공
4. 해당 단계의 Exceptions도 함께 추가
