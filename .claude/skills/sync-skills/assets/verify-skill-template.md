# verify-* 스킬 생성 템플릿

sync-skills의 Step 6에서 새 스킬을 생성할 때 참조합니다.

## 생성할 폴더 구조

새 verify-* 스킬을 생성할 때 **SKILL.md만이 아니라 폴더 구조 전체**를 생성합니다:

```
.claude/skills/verify-<name>/
├── SKILL.md                    # 검증 워크플로우 (필수)
├── references/                 # Claude가 필요할 때 참조하는 문서 (선택)
│   └── (프로젝트에 맞게 추가)
└── assets/                     # 템플릿, 예시 코드 (선택)
    └── (프로젝트에 맞게 추가)
```

### references/ 에 넣을 수 있는 것

Claude가 검증 작업 중 필요할 때 읽는 **참조 문서**입니다:

**코드 프로젝트:**
- **API/라이브러리 문서** — 올바른 사용법, 함수 시그니처
- **코드 패턴 예시** — 올바른 패턴 (이렇게 써야 한다)
- **아키텍처 문서** — 프로젝트 구조/설계 규칙

**문서/기획 프로젝트:**
- **스타일 가이드** — 문서 작성 규칙, 용어집
- **참고 자료** — 외부 규정, 가이드라인, 사양서
- **체크리스트** — 문서 완성도 검증 항목

**데이터 프로젝트:**
- **스키마 정의** — 데이터 구조, 필드 규칙
- **검증 규칙** — 데이터 품질 기준, 허용 범위

### assets/ 에 넣을 수 있는 것

스킬이 **이슈 수정 시 참조하는 템플릿/패턴 예시**입니다:

**코드 프로젝트:**
- **코드 템플릿** — 올바른 보일러플레이트 (이 형태로 고쳐야 한다)
- **설정 파일 예시** — 올바른 설정 형태

**문서/기획 프로젝트:**
- **문서 템플릿** — 기획서, 분석서, 보고서의 표준 구조
- **작성 예시** — 올바른 형식/톤의 실제 예시

**공통:**
- **수정 가이드** — 특정 이슈 유형별 수정 방법

### 사용자 안내

스킬 생성 후 사용자에게 다음을 안내합니다:

```markdown
verify-<name> 스킬이 생성되었습니다.

필요하면 다음 폴더에 참조 문서를 추가할 수 있습니다:
- `.claude/skills/verify-<name>/references/` — API 문서, 코드 스니펫, 스타일 가이드
- `.claude/skills/verify-<name>/assets/` — 코드 템플릿, 수정 가이드

추가한 문서는 SKILL.md의 "참조 파일" 섹션에 링크하면 Claude가 필요할 때 자동으로 참조합니다.
```

## SKILL.md 구조

```yaml
---
name: verify-<name>
description: <트리거 조건을 포함한 설명>. <언제 사용하는지> 후 사용.
---
```

### 필수 섹션

```markdown
# <검증 대상> 검증

## Purpose

<검증 대상>의 규칙 준수를 검증합니다:

1. **<규칙 카테고리 1>** — <설명>
2. **<규칙 카테고리 2>** — <설명>
3. **<규칙 카테고리 3>** — <설명>

## When to Run

- <트리거 상황 1>
- <트리거 상황 2>
- <트리거 상황 3>
- 전체 검증 시 (`/run-skills`를 통해 — PR 전, 콘텐츠 배포 전, 또는 정기 점검)

## 참조 파일

| 파일 | 언제 사용 |
|------|----------|
| [references/<파일명>](references/<파일명>) | <언제 참조하는지> |
| [assets/<파일명>](assets/<파일명>) | <언제 참조하는지> |

(references/나 assets/에 파일이 없으면 이 섹션은 생략)

## Related Files

| File | Purpose |
|------|---------|
| `<실제 파일 경로>` | <이 파일의 역할> |

## Workflow

### Check 1: <검사 이름>

**목적:** <왜 이 검사가 필요한지>

**방법:**

Grep/Glob/Read/Bash를 사용하여 패턴을 탐지합니다.

**PASS:** <통과 조건>
**FAIL:** <실패 조건 + 수정 방법>

## Output Format

| # | 검사 항목 | 상태 | 상세 |
|---|----------|------|------|
| 1 | <검사 1> | PASS/FAIL | <상세> |

## Exceptions

다음은 **문제가 아닙니다:**

1. **<예외 1>** — <왜 위반이 아닌지>
2. **<예외 2>** — <왜 위반이 아닌지>
```

## 이름 규칙

- 반드시 `verify-` 접두사 사용
- kebab-case (예: `verify-error-handling`)
- 사용자가 접두사 없이 이름을 제공하면 자동 추가

## 생성 후 필수 업데이트

1. `sync-skills/SKILL.md` — 등록 테이블에 행 추가
2. `run-skills/SKILL.md` — 실행 대상 테이블에 행 추가
3. `CLAUDE.md` — Skills 테이블에 행 추가
