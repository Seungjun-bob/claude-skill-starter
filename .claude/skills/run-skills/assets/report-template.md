# 통합 검증 보고서 형식

run-skills의 Step 3에서 보고서를 생성할 때 이 형식을 사용합니다.

---

## 모든 검증 통과 시

```markdown
## 구현 검증 보고서

모든 검증을 통과했습니다!

| 검증 스킬 | 상태 | 검사 항목 |
|-----------|------|---------|
| verify-<name1> | PASS | N개 |
| verify-<name2> | PASS | N개 |

작업물이 프로젝트의 모든 규칙을 준수합니다.
```

## 이슈 발견 시

```markdown
## 구현 검증 보고서

### 요약

| 검증 스킬 | 상태 | 이슈 수 |
|-----------|------|---------|
| verify-<name1> | PASS | 0 |
| verify-<name2> | X개 이슈 | N |

**발견된 총 이슈: X개**

### 상세 이슈

| # | 스킬 | 파일 | 문제 | 수정 방법 |
|---|------|------|------|-----------|
| 1 | verify-<name> | `<파일경로>:<줄번호>` | <문제 설명> | <수정 방법> |
| 2 | verify-<name> | `<파일경로>:<줄번호>` | <문제 설명> | <수정 방법> |
```

## 스킬별 진행 상황

각 스킬 실행 완료 후:

```markdown
### verify-<name> 검증 완료

- 검사 항목: N개
- 통과: X개
- 이슈: Y개
- 면제: Z개
```

## 수정 후 재검증

```markdown
## 수정 후 재검증

| 검증 스킬 | 수정 전 | 수정 후 |
|-----------|---------|---------|
| verify-<name1> | X개 이슈 | PASS |
| verify-<name2> | Y개 이슈 | PASS |

모든 검증을 통과했습니다!
```
